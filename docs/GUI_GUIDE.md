# GUI Guide

## Purpose
The GUI provides a browser-based workflow for preprocessing FAUST meshes, training models, monitoring progress, generating reports, and downloading artifacts.

## Quick Start
1. Place FAUST mesh files in `data/raw/`, or upload them from the browser.
2. Run `bash start.sh`.
3. Open `http://localhost:8080`.

## Supported Tasks
Classification models:

- `mlp`
- `cnn1d`
- `pointnet`

Autoencoder models:

- `mlp_ae`
- `pointnet_ae`

## Workflow
1. Upload raw mesh files or reuse files already present in `data/raw/`.
2. Reuse an existing processed cache or configure preprocessing:
   - `num_points`
   - `samples_per_mesh`
   - centering
   - scale normalization
   - rotation range
   - translation range
3. Choose task type:
   - Classification
   - Autoencoder
4. Set training parameters:
   - batch size
   - epochs
   - learning rate
   - dropout
5. Start training and monitor live curves.
6. Generate an evaluation report with task-specific metrics.
7. Download the checkpoint and report artifact.

## Current Defaults
The GUI now matches the active project config:

- `num_points = 500`
- `normalize_center = true`
- `normalize_scale = false`
- `augmentation.normalize = false`
- `rotation_range = 360`
- `translation_range = 0.0`
- classification dropout default = `0.5`
- autoencoder dropout default = `0.1`

## Notes
- Classification shows loss and accuracy curves.
- Autoencoder training focuses on reconstruction loss.
- The evaluation stage renders backend metrics directly: accuracy / precision / recall / F1 for classification, Chamfer Distance for autoencoders.
- Downloaded reports are JSON artifacts generated by the backend.
- The GUI is a convenience layer over the same training code used by the CLI.

## Requirements

- Docker and Docker Compose
- 8GB+ RAM recommended
