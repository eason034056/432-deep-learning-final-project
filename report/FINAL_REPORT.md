## 1. Abstract

This project addresses two cognitive problems using the FAUST point cloud dataset: (1) **human identification** from sparse 3D point clouds (10-class classification), and (2) **point cloud compression/reconstruction** via autoencoders. We implement and compare multiple deep learning architectures: MLP, 1D-CNN, and PointNet++ for classification; MLP-based and PointNet++-based autoencoders for compression. Our best classification model (PointNet++) achieves approximately 70–85% validation accuracy with permutation invariance. For compression, the PointNet++ autoencoder outperforms the MLP baseline in Chamfer Distance. We provide a deployment architecture with Docker, Flask API, and web GUI, along with a model maintenance and parameter update plan.

**Keywords**: Point clouds, mmWave radar, human identification, PointNet++, autoencoder, Chamfer Distance, FAUST dataset.

## 2. Exploratory Data Analysis

### 2.1 Data Summary

- **Dataset**: FAUST (Faces of Articulated Super-humans Through Training)
- **Record counts**: 100 mesh files (10 subjects × 10 poses), augmented to 10,000 samples (100 samples per mesh)
- **Schema**: Each sample is a point cloud of shape (200, 3) — 200 points with (x, y, z) coordinates
- **Missing values**: None (preprocessed mesh data)
- **Value ranges**: Normalized to unit sphere; coordinates typically in [-1, 1]

### 2.2 Visualizations

The EDA notebook (`notebooks/eda.ipynb`) includes:

- **Bar chart**: Class distribution (samples per subject)
- **3D scatter plots**: Sample point clouds from different subjects
- **Histograms**: Univariate distribution of x, y, z coordinates
- **Violin plots**: Coordinate distributions by subject
- **Heatmap**: Correlation of mean coordinates per sample
- **Q-Q plots**: Normality check of coordinate distributions
- **Outlier analysis**: Spread and max distance from centroid

### 2.3 Key Findings

- Data is balanced across 10 subjects
- Point clouds are normalized to unit sphere; no significant missing values
- Some samples show higher spread (potential outliers) — may affect model robustness

## 3. Model Training and Evaluation

### 3.1 Problem 1: Human Identification (Classification)

**Task**: 10-class classification — identify which of 10 subjects a point cloud belongs to.

**Models (Champion-Challenger)**:

| Model | Architecture | Pros | Cons |
|-------|--------------|------|------|
| **MLP Baseline** | Flatten → FC layers | Fast, simple | Order-dependent, low accuracy (~20–40%) |
| **1D-CNN** | Conv1D + GlobalMaxPool | Captures local patterns | Partially order-invariant (~65–70%) |
| **PointNet++ (Champion)** | Set Abstraction + hierarchical | Permutation-invariant, best accuracy (~70–85%) | Longer training |

**Evaluation Metrics**: Accuracy, F1-Score (macro/micro), Precision, Recall, Confusion Matrix, **ROC-AUC** (one-vs-rest, macro-averaged).

**Results**: PointNet++ achieves the best validation accuracy. ROC curves and AUC are generated by `src/evaluate.py`.

### 3.2 Problem 2: Point Cloud Compression (Autoencoder)

**Task**: Encode point clouds to a low-dimensional latent space and reconstruct them.

**Models (Champion-Challenger)**:

| Model | Encoder | Decoder | Pros | Cons |
|-------|---------|---------|------|------|
| **MLP AE (Challenger)** | Flatten → FC | FC → reshape | Simple, fast | Order-dependent |
| **PointNet++ AE (Champion)** | Set Abstraction | Folding (latent + grid → MLP) | Permutation-invariant, better geometry | More parameters |

**Evaluation Metrics**: Chamfer Distance (CD), Earth Mover's Distance (optional), reconstruction visualization.

**Results**: PointNet++ autoencoder typically achieves lower Chamfer Distance than the MLP baseline.

### 3.3 Algorithm Pros and Cons

- **MLP**: Simple but ignores point order and geometry; good baseline.
- **1D-CNN**: Learns local patterns; sensitive to point ordering (depends on sorting).
- **PointNet++**: Permutation-invariant, hierarchical features; state-of-the-art for point clouds but more compute-intensive.

## 4. Model Operations

### 4.1 Deployment Architecture

- **Data flow**: Raw point clouds → Preprocessing → Model inference → Predictions
- **Deployment**: Flask REST API + Web GUI; containerized with Docker Compose
- **Access**: `http://localhost:8080` after running `bash start.sh`

See `docs/MODEL_OPS.md` for detailed architecture diagrams and component descriptions.

### 4.2 Model Maintenance Plan

- **Monitoring**: Validation accuracy (classification), Chamfer Distance (AE), inference latency
- **Retraining triggers**: Scheduled (quarterly), performance drop, new subjects, data drift
- **Pipeline**: Data collection → Preprocessing → Training → Validation → Deployment → Rollback if needed

### 4.3 Parameter Update Process

- **Config**: `config.yaml` for hyperparameters; versioned in Git
- **A/B testing**: Train challenger → Evaluate → Shadow deploy → Promote if better
- **Rollback**: Restore previous checkpoint; document incident

## 5. Conclusion

We implemented a deep learning platform for mmWave radar-based human identification and point cloud compression. Two cognitive problems were addressed: (1) classification with MLP, 1D-CNN, and PointNet++, and (2) compression with MLP and PointNet++ autoencoders. PointNet++ consistently outperforms baselines due to its permutation invariance and hierarchical structure. We provided a deployment architecture, maintenance plan, and parameter update process to support production use.

**Future work**: Temporal modeling (e.g., Bi-LSTM) for sequential point clouds, larger datasets, and real mmWave radar deployment.
